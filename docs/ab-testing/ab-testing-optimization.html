TL;DR

Traditional significance testing uses p-values and statistical power to control error rates — typically you’ll see a False Positive rate of 5% and a False Negative rate of 20%, implying false positives are four times costlier. In industry, that assumption rarely holds.

Our goal is to maximize value and successful customer outcomes — so why anchor decisions to arbitrary error thresholds borrowed from academic research? This paper sets aside those tools entirely and asks a different question: what stopping rule actually maximizes long-term outcomes?

A Simulation Framework

Traditional power analysis works forward: specify α, β, and an MDE, and it returns a required sample size.

First, it assumes a single, known effect size — but in practice, test effects follow a distribution; some experiments move the needle, most don’t, some backfire. Second, it outputs a sample size with no regard for what happens after the test ends or how many users will be exposed to the winner.

This framework inverts that. Specify the test population, the post-test audience, and the distribution of effects you realistically expect — and it returns the stopping rule that maximizes long-term outcomes given that context.

Total users in the experiment and their control/treatment split. The first 7 days are burn-in — no stopping decisions. The experiment runs at most 20 days.
The population exposed to the winner after the decision is made. More post-test traffic means more value at stake — potentially justifying a more aggressive stopping rule.
The realistic lift you expect comes from a distribution of possible effects — not a single number. Here, experiments have an equal chance of being good or bad.

Optimal Stopping Algorithm

For each of 100,000 simulated experiments and each significance level α ∈ [0.01, 0.50]:

  1. Burn-in: No stopping decisions before min_test_days = 7
  2. Daily check: Stop when either condition’s Pr(Best) exceeds 1 - α
  3. Strategy: If neither condition breaches by the final day:
      1. Select Best: roll out whichever condition is currently ahead
      1. Retain Control: keep control
  4. Rollout: Deploy the winner to remaining test users and post-test traffic
  5. Evaluate: Pick the α with the highest average long-term outcome

Results

Each chart plots the projected mean outcome against the stopping threshold α. The peak of the curve is the optimal rule for that scenario. Use the dropdowns below to vary the baseline conversion rate, post-test population, and effect distribution — and watch how the optimal threshold shifts.

Insights

P-hacking is good for business. The optimal stopping threshold is well below the conventional α = 0.05 — stopping early on a signal, rather than waiting for statistical certainty, consistently produces better long-term outcomes in simulation.

import simulationengine as se

# ── Experiment Assumptions ─────────────────────────────────────────
CONTROL_MEAN       = 0.10      # pre-experiment conversion rate
MAX_TEST_N         = 100_000   # total users in the experiment
SPLIT              = 0.50      # treatment allocation (50/50)
MIN_TEST_DAYS      = 7         # mandatory burn-in before stopping decisions
NUMBER_TEST_DAYS   = 20        # maximum experiment duration
POST_TEST_N        = 500_000   # users exposed to winner once MAX_TEST_N is reached
NUMBER_SIMULATIONS = 100_000   # simulations to run
SEED               = 42
ITC_MEAN           = 100       # index-to-control mean
ITC_SE             = 1         # index-to-control standard error

df = se.create_simulation_df(
    number_test_days=NUMBER_TEST_DAYS,
    number_simulations=NUMBER_SIMULATIONS,
    seed=SEED,
    condition_parameters=[
        {
            "mean": CONTROL_MEAN,
            "trials": int(MAX_TEST_N * SPLIT),
        },
    ],
)

random_effect_parameters_0 = {
    "itc_mean": ITC_MEAN,
    "itc_se": ITC_SE,
    "trials": int(MAX_TEST_N * SPLIT),
}

df = se.add_random_effect_condition(df, random_effect_parameters_0)
df = se.add_cumulative_columns(df)
df = se.compute_cumulative_std_rates(df)
df = se.compute_delta_stats(df)
df = se.compute_test_stats(df)
df = se.calculate_remaining_test_potential(df)
df = se.calculate_launch_potential(df, post_test_n=POST_TEST_N)
df = se.calculate_total_potential(df)

## Optimal Stopping Algorithm

For each of 100,000 simulated experiments and each significance level α ∈ [0.01, 0.50]:

1. **Burn-in:** No stopping decisions before `min_test_days = 7`
2. **Daily check:** Stop when either condition's Pr(Best) exceeds `1 - α`
3. **Strategy:** If neither condition breaches by the final day:
   - A) _Select Best_: roll out whichever condition is currently ahead
   - B) _Retain Control_: keep control
4. **Rollout:** Deploy the winner to remaining test users and post-test traffic
5. **Evaluate:** Pick the α with the highest average long-term outcome

## Results

Each chart plots the projected mean outcome against the stopping threshold α. The peak of the curve is the optimal rule for that scenario. Use the dropdowns below to vary the baseline conversion rate, post-test population, and effect distribution — and watch how the optimal threshold shifts.
